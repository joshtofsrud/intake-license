# DEPLOY — complete drop-in

This zip is a full replacement for `/var/www/intake` application files.
It contains the fixes for admin login, signup auto-login, and URL defaults,
plus the `app/helpers.php` file that went missing from the server and caused
the current 500 error.

---

## What's in it

Everything the app needs that belongs in git — controllers, models, views,
routes, migrations, config, public entry point, composer.json, bootstrap.

## What's NOT in it

- `vendor/` — regenerated by `composer install`
- `node_modules/` — regenerated by `npm install` if you use it
- `.env` — yours stays on the server, untouched
- `.git/` — your server's git history is preserved
- `storage/logs/*`, `storage/framework/cache/*`, `storage/framework/sessions/*`,
  `storage/framework/views/*` — zip contains only empty directories with
  `.gitignore` files. Your existing sessions and logs are NOT overwritten.

---

## Extraction

All paths in the zip are relative to the app root. Extract directly into
`/var/www/intake/`:

```bash
# On your laptop:
scp intake-complete.zip root@142.93.50.209:/tmp/

# On the server:
ssh root@142.93.50.209
cd /var/www/intake
unzip -o /tmp/intake-complete.zip
```

`unzip -o` overwrites files that ARE in the zip but leaves everything else
(vendor/, .env, storage contents, .git, custom server-side files) alone.

## Post-deploy

```bash
cd /var/www/intake

# Fix ownership (extraction is done as root, app runs as www-data)
sudo chown -R www-data:www-data /var/www/intake

# Rebuild Composer autoload — this is what fixes the current 500
sudo -u www-data composer install --no-dev --optimize-autoloader

# Clear all Laravel caches (safe — they're rebuilt below)
sudo -u www-data php artisan config:clear
sudo -u www-data php artisan route:clear
sudo -u www-data php artisan view:clear
sudo -u www-data php artisan cache:clear

# Run any new migrations (including the master-admin users table migration)
sudo -u www-data php artisan migrate --force

# Rebuild caches for production
sudo -u www-data php artisan config:cache
sudo -u www-data php artisan route:cache

# Reload PHP-FPM to drop opcache
sudo systemctl reload php8.3-fpm
```

## If something's still wrong

Check the Laravel log:
```bash
tail -100 /var/www/intake/storage/logs/laravel-$(date +%Y-%m-%d).log
```

And PHP-FPM/Nginx:
```bash
tail -50 /var/log/nginx/error.log
```

Paste whichever has recent errors.
